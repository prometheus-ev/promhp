h3. Entwicklungsplatformen

* Die meisten unserer Server und Entwicklungs-Machinen laufen unter GNU/Linux, <%=t 'namely', 'namentlich' %> "Debian":http://debian.org/ <%=t 'and', 'und' %> "Red Hat Enterprise Linux":http://redhat.com/rhel/

* "Github":https://github.com - ein webbasierter Hosting-Dienst für Software-Entwicklungsprojekte. Er verwendet namensgebenderweise das Versionsverwaltungs-System _Git_.


h3. Programmiersprachen

* "Ruby":http://ruby-lang.org - eine höhere Programmiersprache, die Mitte der neunziger Jahre von dem Japaner Yukihiro Matsumoto entworfen wurde. Ruby ist interpretiert und objektorientiert, unterstützt aber mehrere weitere Programmierparadigmen (unter anderem prozedurale und funktionale Programmierung sowie Nebenläufigkeit), bietet dynamische Typisierung, Reflexion und automatische Speicherbereinigung.


h3. Entwicklungswerkzeuge

* "Git":http://git-scm.com - eine freie Software zur verteilten Versionsverwaltung von Dateien, die ursprünglich für die Quellcode-Verwaltung des Linux-Kernels entwickelt wurde.

* "Subversion":http://subversion.apache.org - eine freie Software zur Versionsverwaltung von Dateien und Verzeichnissen. Entwickler verwenden Subversion für den Zugriff auf aktuelle und vergangene Versionen von Dateien, sei es Quellcode, Internetseiten oder Dokumentation.

* "Redmine":http:www.redmine.org - ein webbasiertes Projektmanagement-Tool. Es kann für Benutzer- und Projektverwaltung, Diskussionsforen, Wikis, zur Ticketverwaltung oder Dokumentenablage genutzt werden. Weltweit wird Redmine bei mehreren großen Projekten verwendet, unter anderem bei dem Issue-Tracking-System der Programmiersprache Ruby oder dem Entwicklerportal von TYPO3.

h3. Bibliotheken

* "ImageMagick":http://www.imagemagick.org - eine Open-Source Sammlung von Programmen zur Anzeige, Konvertierung und Anpassung von Rasterbildern. ImageMagick ist in der Lage mehr als 100 Dateiformate zu lesen und schreiben.

* "LibXML Ruby":https://github.com/xml4r/libxml-ruby - Ruby Sprachanbindung für das GNOME's Libxml2 XML Toolkit.

* "Ferret":https://github.com/dbalmain/ferret - ein Ruby Port der "Java Lucene":http://jakarta.apache.org/lucene Suchmachine.

* "Lingo":http://lex-lingo.de - ein frei verfügbares System zur linguistisch und statistisch basierten automatischen Indexierung des Deutschen und Englischen. Lingo arbeitet mit einem Wörterbuchmodell, das eine Grundformidentifizierung bzw. -reduktion mit einem Grundformenwörterbuch und einer zugehörigen einfachen Suffixliste realisiert. Lingo verfügt zusätzlich über eine algorithmische Kompositumzerlegung, eine algorithmische und lexikalische Mehrworterkennung sowie eine allgemeine lexikalische Relationierung.


h3. Web Applications

* "Apache":http://httpd.apache.org - ein Open Source HTTP Server für moderne Betriebssysteme, inklusive UNIX und Windows NT. Ziel des Projektes ist die Entwicklung eines sicheren, effizienten und erweiterbaren Servers zur Bereitstellung von HTTP Diensten in Übereinstimmung mit aktuellen HTTP Standards.

* "Ruby on Rails":http://rubyonrails.org - ein in der Programmiersprache Ruby geschriebenes und quelloffenes Web Application Framework. Rail ist ein "full-stack framework", was bedeutet dass es den Entwickler in die Lage versetzt Informationen über den Webserver zu erhalten, Datenbankabfragen durchzuführen und neue Inhalte unter Zuhilfenahme von Vorlagen darzustellen.

* "Mongrel":https://github.com/evan/mongrel - ein Open Source HTTP Server (bzw. eine Bibliothek), typischerweise verwendet um in Ruby geschriebene Web Applications auszuführen und gleichzeitig eine Standard HTTP Schnittstelle zu präsentieren. Dieser Ansatz ermöglicht die Hintereinanderschaltung von Servern, z.B. das Voranstellen eines Web Proxy, eines Load Balancer, oder gar einer Kombination von beiden.

* "PmWiki":http://www.pmwiki.org - ein Wiki-basiertes System zur gemeinschaftlichen Erstellung und Verwaltung von Websites.
